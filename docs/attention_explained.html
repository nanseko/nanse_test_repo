0배경: CUT 모델은 무엇인가

Attention을 어디에 넣었는지 이해하려면, 먼저 CUT Generator의 데이터 흐름을 알아야 합니다.

CUT은 짝이 없는(unpaired) 영상 변환을 위한 모델로, CycleGAN과 달리 한 방향(one-sided) 변환만으로 대조 학습(contrastive learning) 기반의 PatchNCE Loss를 사용해 입력·출력 패치의 대응 관계를 학습합니다.

Generator는 전형적인 ResNet 기반 인코더–변환–디코더 구조입니다.

Encoder

7×7 ConvBlock → 2회 Downsampling. 입력 영상을 압축된 특징 공간(256ch, 64×64)으로 인코딩.

ResBlocks ×9

9개의 Residual Block에서 도메인 변환(SAR → Optical)의 핵심 표현 변환이 일어남.

Decoder

2회 Upsampling → 7×7 Conv(tanh). 특징을 다시 영상으로 복원.

핵심 연결고리 — PatchNCE는 Generator 내부 특징을 그대로 읽는다.
Encoder(generator, nce_layers=[0,3,5,7,11]) 는 Generator의 중간 레이어 출력을 그대로 가져와 대조 학습에 사용합니다. 즉 Generator 중간 특징의 품질 = PatchNCE 학습 신호의 품질이며, 이것이 Attention을 "어디에" 넣을지 결정하는 가장 중요한 단서입니다. (4장에서 상술)

1왜 Attention을 추가하는가

대상 작업은 SAR → Optical(광학) 영상 변환입니다. 이 작업의 고유한 어려움이 Attention 도입의 근거입니다.

🛰️
SAR의 speckle 잡음
SAR 영상은 곱셈성 speckle 잡음이 강합니다. 일반 Conv는 잡음과 신호를 구분하지 못해 speckle이 그대로 광학 텍스처로 "번역(hallucination)"되기 쉽습니다. Attention은 의미 있는 영역에 가중치를 주어 잡음 억제를 돕습니다.
🧭
강한 구조·방향성 정보
도로, 건물 경계, 해안선, 농경지 패턴 등 위치·방향 구조가 중요합니다. 특히 좌표(가로/세로) 방향성을 잘 보존해야 하므로 Coordinate Attention이 적합합니다.
🎯
채널·공간 선택성 부족
기본 CUT의 Conv는 모든 채널·위치를 동등하게 취급합니다. 어떤 채널/영역이 도메인 변환에 중요한지 명시적으로 강조하면 변환 품질이 올라갑니다 → CBAM.
🔗
PatchNCE 신호 강화
Attention으로 정제된 특징을 PatchNCE가 읽으면, 대조 학습의 양/음 패치 구분이 또렷해져 구조 보존(structure preservation)이 강화됩니다. 손실 함수 수정 없이 성능 향상이 가능합니다.
한 줄 요약: Attention은 ① speckle 잡음 억제, ② 구조/방향성 보존, ③ 중요 채널·영역 강조, ④ PatchNCE 신호 정제를 통해 기존 CUT 구조와 손실 함수를 건드리지 않고 SAR-to-Optical 변환 품질을 끌어올리기 위해 추가되었습니다.

3어디에 추가했는가 — 3개의 삽입 지점

Attention은 Generator(modules/cut_model.py)의 세 구간에 선택적으로 삽입할 수 있습니다. 각 구간은 독립 플래그로 켜고 끕니다.

플래그삽입 위치구현 위치
attention_encoder ① 초기 7×7 ConvBlock 뒤
② 첫 downsampling block 뒤
③ 두 번째 downsampling block 뒤		 cut_model.py · maybe_attention(...) (name=attention_encoder_0/1/2)
attention_resblocks 각 ResBlock의 residual branch (conv 2개 통과 후, skip 더하기 전) layers.py · ResBlock · attention_position='residual'
attention_decoder decoder의 각 upsampling block 뒤 cut_model.py · (name=attention_decoder_0/1)

3-1. 실제 삽입 코드 (Encoder 예시)

x = ConvBlock(64, 7, ...)(x)
if attention_encoder:
    x = maybe_attention(x, attention_type, attention_reduction, name='attention_encoder_0')  # ① 7x7 뒤
...
x = ConvBlock(128, 3, ...)(x)
if attention_encoder:
    x = maybe_attention(x, ..., name='attention_encoder_1')   # ② 1st down 뒤
x = AntialiasSampling(4, mode='down', impl=impl)(x)
x = ConvBlock(256, 3, ...)(x)
if attention_encoder:
    x = maybe_attention(x, ..., name='attention_encoder_2')   # ③ 2nd down 뒤

3-2. ResBlock 내부 삽입 (residual branch)

def call(self, inputs):
    x = self.conv_block1(self.reflect_pad1(inputs))
    x = self.conv_block2(self.reflect_pad2(x))

    if self.attention_position == 'residual':   # 기본값
        x = self.attention(x)        # ← residual 가지에만 Attention 적용
        return inputs + x            #   그 후 skip connection
    # 'post_add' 옵션: skip 더한 전체 출력에 적용
    x = inputs + x
    return self.attention(x)
Generator는 ResBlock을 기본값 attention_position='residual' 로 사용합니다. 이는 원본 항등 경로(skip)를 보존하면서 변환 신호(residual)에만 Attention을 적용하기 위함입니다(자세한 이유는 4장).

4왜 그 위치인가 — 가장 중요한 핵심

단순히 "Attention을 넣었다"가 아니라, 각 위치가 갖는 의미가 SAR-to-Optical 변환과 PatchNCE 학습에 직결됩니다.

🔑 핵심 논리: PatchNCE가 읽는 바로 그 특징을 정제한다

CUT의 nce_layers=[0,3,5,7,11] 는 Generator의 인코더~초반 ResBlock 출력을 대조 학습 패치로 사용합니다. 따라서 Attention을 인코더와 ResBlock에 넣으면:

  • PatchNCE가 Attention으로 정제된 특징을 자동으로 사용하게 됨 → 양/음 패치 대비가 또렷해짐
  • 손실 함수(losses.py)는 한 줄도 수정할 필요가 없음 — 특징 품질만 올려 학습 신호를 강화
  • 구조 보존(도로·경계·해안선)이 대조 학습 단계에서부터 강화됨

① Encoder 위치

왜 여기? 입력 SAR의 speckle 잡음이 가장 먼저 들어오는 곳. 여기서 중요한 구조/채널을 강조하고 잡음을 억제하면, 이후 모든 단계와 PatchNCE 패치(layer 0,3)가 깨끗한 특징을 받습니다.

→ 잡음 억제 + 초기 구조 강조의 최적 지점.

② ResBlock(residual) 위치

왜 여기? 도메인 변환(SAR→Optical)의 핵심 표현 변환이 일어나는 곳. residual 가지에만 적용해 원본 항등 경로는 보존하면서 변환 성분만 선택적으로 강조 → 학습 안정성 유지.

→ nce_layers 5,7,11 이 이 영역을 읽어 NCE 신호가 직접 강화됨.

③ Decoder 위치 (선택)

왜 선택적? 출력 영상의 외형(색·질감)에 직접 영향을 줌. 효과가 크지만 잘못 쓰면 artifact 위험.

→ encoder+resblocks로 먼저 검증 후 추가 권장.

요약 — 위치 선택의 원칙
Attention은 ① 잡음이 유입되는 인코더, ② 변환이 일어나며 동시에 PatchNCE가 읽는 ResBlock에 우선 배치합니다. 이는 "특징 품질 향상 → 대조 학습 신호 강화 → 구조 보존 개선"이라는 인과 사슬을 노린 것이며, 디코더는 출력에 직접 작용하므로 검증 후 선택적으로만 추가합니다.

이 그림이 핵심입니다. Attention은 Generator 내부 특징을 바꾸고, PatchNCE는 그 특징을 자동으로 읽습니다. 따라서 Attention 추가만으로 대조 학습 신호까지 함께 강화됩니다.

7-1. 평가 시 함께 볼 항목 (SAR-to-Optical)

  • 구조 보존: 도로·건물 경계·해안선·농경지 패턴이 유지되는가
  • 색 변환 품질: optical 도메인의 색감이 자연스러운가
  • texture hallucination: SAR speckle이 과도하게 광학 텍스처로 번역되지 않는가
  • mode collapse: 다양한 입력이 비슷한 결과로 수렴하지 않는가
  • NCE 안정성: Attention 추가 후 학습 초반 loss가 급격히 불안정해지지 않는가

정량 평가는 paired GT가 없으므로 FID/KID, downstream segmentation 성능 등을 별도 구성합니다.
