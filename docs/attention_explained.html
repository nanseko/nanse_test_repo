CUT 생성기에 Attention을 추가한 이유와 위치

0먼저: CUT 생성기는 어떻게 생겼나

Attention을 "어디"에 넣는지 이해하려면, 생성기(Generator)의 3단 구조를 먼저 알아야 합니다.

① Encoder

입력 SAR을 점점 작은 "특징(feature)"으로 압축. 7×7 Conv → 2번 다운샘플 (256채널, 64×64).

② ResBlock ×9

압축된 특징 공간에서 도메인 변환(SAR→Optical)의 핵심 변형이 일어나는 9개 블록.

③ Decoder

특징을 다시 이미지로 복원. 2번 업샘플 → 7×7 Conv(tanh).

비유: Encoder=재료를 잘게 다지기, ResBlock=요리(맛 변환), Decoder=접시에 담기. Attention은 이 과정 곳곳에서 "중요한 재료/부분에 더 집중하라"고 지시하는 양념입니다.

1Attention이란? — "중요한 곳을 비추는 스포트라이트"

Attention은 특징맵의 각 위치/채널에 0~1 가중치를 곱해 중요한 곳은 키우고 덜 중요한 곳은 줄이는 장치입니다.

원본 특징 (잡음 섞임)
× 가중치 =
Attention 적용 후
현재: OFF
왼쪽엔 의미 없는 잡음(speckle)이 섞여 있습니다. Attention을 켜면 중요한 영역(밝은 곳)은 유지·강조되고 잡음(어두운 잡티)은 억제됩니다. 이게 Attention의 본질 — "선택적 집중"입니다.

2왜 CUT에 Attention을 추가하나 (SAR→Optical의 어려움)

대상 작업은 SAR(레이더) 영상을 Optical(광학) 영상으로 바꾸는 것. 이 작업만의 어려움이 곧 Attention 도입의 이유입니다.

🛰️ ① speckle 잡음

SAR엔 알갱이 같은 잡음이 강합니다. 일반 Conv는 잡음·신호를 구분 못해 잡음을 그대로 "광학 질감"으로 잘못 번역합니다. Attention이 잡음을 억제합니다.

🧭 ② 구조·방향성

도로, 건물 경계, 해안선, 농경지처럼 위치·방향이 중요한 구조가 많습니다. 방향성을 잡는 Coordinate Attention이 잘 맞습니다.

🎯 ③ 채널·영역 선택

어떤 특징 채널/영역이 변환에 중요한지 일반 Conv는 모두 동등 취급. CBAM이 중요 채널·영역을 명시적으로 강조합니다.

🔗 ④ PatchNCE 신호 강화

Attention으로 다듬은 특징을 PatchNCE가 읽으면 대조 학습 신호가 또렷해져 구조 보존이 강해집니다. 손실 함수는 그대로!

정리: Attention은 ①잡음 억제 ②구조/방향 보존 ③중요 채널·영역 강조 ④PatchNCE 신호 정제를 통해, 기존 CUT 구조·손실을 거의 그대로 둔 채 SAR→Optical 품질을 끌어올리려는 시도입니다.

3Attention 종류 ① CBAM — 채널 + 공간 (애니메이션)

CBAM은 "어떤 채널이 중요한가"를 먼저, 그다음 "어떤 위치가 중요한가"를 순서대로 강조합니다. 아래 ▶ 재생을 눌러 단계를 보세요.

① 채널별 중요도 (GAP/GMP→MLP)→ ② 채널 가중치 곱→ ③ 공간 중요도 (7×7 Conv)→ ④ 공간 가중치 곱
채널 Attention (채널별 0~1 가중치)
공간 Attention (위치별 0~1 가중치)
재생 버튼을 눌러보세요.
코드: CBAM = ChannelAttention(GAP+GMP→공유 MLP→sigmoid) 적용 후 SpatialAttention(채널평균/최댓값→7×7 Conv→sigmoid) 적용. 입력과 같은 shape을 반환해 그대로 끼워넣습니다.

4Attention 종류 ② Coordinate Attention — 방향성 (애니메이션)

가로(W)·세로(H) 방향을 따로 요약해서, "어느 행/열이 중요한지"를 잡습니다. 도로·해안선 같은 방향 구조 보존에 강합니다.

세로(H) 방향 중요도 가로(W) 방향 중요도
Coordinate Attention은 H방향 pooling과 W방향 pooling을 각각 구해 행·열의 중요도를 만들고, 이를 곱해 "특정 방향의 구조"를 강조합니다. SAR→Optical 1순위 추천 이유입니다.
코드: CoordinateAttention = H/W 방향 평균 pooling → concat → 1×1 bottleneck Conv → 다시 H/W로 분리 → 각각 1×1 Conv + sigmoid → 입력 × attn_h × attn_w.

6★ 왜 "그 위치"인가 — 가장 중요한 핵심

단순히 넣은 게 아니라, 각 위치에는 분명한 이유가 있습니다. 핵심은 PatchNCE와의 연결입니다.

🔑 핵심: PatchNCE는 "생성기 중간 특징"을 그대로 읽는다

CUT의 PatchNCE는 생성기의 인코더~ResBlock 초반 출력(nce_layers)을 대조 학습에 사용합니다. 따라서 그 지점에 Attention을 넣으면:

  • PatchNCE가 Attention으로 정제된 특징을 자동으로 읽음 → 정답/오답 패치 구분이 또렷해짐
  • 손실 함수(losses.py)는 한 줄도 안 고침 — 특징 품질만 올려 학습 신호를 강화
  • 구조 보존(도로·경계·해안선)이 대조 학습 단계부터 강해짐
Attention 정제 PatchNCE로 흐르는 정제 특징

① Encoder 위치

왜? SAR 잡음이 가장 먼저 들어오는 입구. 여기서 중요한 구조를 강조하고 잡음을 억제하면 이후 모든 단계 + PatchNCE 패치가 깨끗한 특징을 받습니다.

② ResBlock 위치

왜? 도메인 변환의 핵심이 일어나는 곳. residual 가지에만 적용해 원본 경로는 보존하며 변환 성분만 강조 → 안정적. PatchNCE도 이 영역을 읽어 신호가 직접 강화됨.

③ Decoder 위치 (선택)

왜 선택? 출력 외형(색·질감)에 직접 작용. 효과는 크지만 잘못 쓰면 artifact 위험 → encoder+resblocks 검증 후 추가 권장.

위치 선택 원칙: ① 잡음이 들어오는 인코더, ② 변환이 일어나며 동시에 PatchNCE가 읽는 ResBlock에 우선 배치. "특징 품질↑ → 대조 학습 신호↑ → 구조 보존↑"의 인과 사슬을 노린 것. 디코더는 출력에 직접 작용하므로 검증 후 선택적으로만.

7ResBlock 안에서는 정확히 어디? (residual vs post_add)

ResBlock은 "원본을 그대로 더하는 길(skip)"과 "변형하는 길(residual)"로 나뉩니다. Attention을 어느 쪽에 거느냐가 중요합니다.

✅ residual (기본값)

x = conv2(conv1(input))   # 변형 경로
x = attention(x)          # 변형분에만 적용
return input + x          # 원본 보존 후 더함

원본(input)은 그대로 두고 변형 성분에만 Attention. 학습이 안정적이라 기본값으로 채택.

post_add (선택)

x = input + conv2(conv1(input))
return attention(x)       # 합쳐진 전체에 적용

skip까지 더한 전체 출력에 Attention. 더 공격적이지만 원본 경로도 눌릴 수 있어 주의.

코드: ResBlock(attention_position='residual') 가 기본. 'post_add' 로 바꾸면 전체 출력에 적용됩니다.

9한 장 요약

  • Attention = 특징에 0~1 가중치를 곱해 중요한 곳 강조, 잡음 억제하는 스포트라이트.
  • CBAM = 채널+공간 강조 / Coordinate = 가로·세로 방향 강조(SAR 구조에 적합).
  • 어디에: 인코더(3곳) · ResBlock residual · 디코더(선택). 플래그로 개별 On/Off.
  • 왜 그 위치: 잡음이 들어오는 인코더 + 변환이 일어나고 PatchNCE가 읽는 ResBlock에 두면, 손실 함수 수정 없이 대조 학습 신호까지 강해짐.
  • 기존 CUT 골격·손실은 유지 — Attention은 사이에 끼워넣기만 함.
