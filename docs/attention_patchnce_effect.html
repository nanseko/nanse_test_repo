ATTENTION ↔ PATCHNCE · 초보자용

Attention은 PatchNCE에 어떤 영향을 줄까?

① tap(=PatchNCE가 읽는 지점)에 Attention을 넣으면 무슨 일이 일어나는지, ② 반대로 tap이 없는 곳(디코더 등)에 Attention을 넣으면 어떤 효과가 있는지를 움직이는 그림과 직접 만지는 도표로 설명합니다.

0. tap 복습 1. tap에 Attention ▶ 2. 왜 또렷해지나 3. tap 밖 Attention ▶ 4. 두 경로 ▶ 5. 위치별 정리 ▶ 6. 요약

0복습: PatchNCE가 읽는 곳은 5군데뿐

PatchNCE는 생성기 전체가 아니라 정해진 5개 지점(tap)의 특징만 패치로 뽑아 비교합니다.

tap = 입력픽셀 · enc_conv1 · enc_conv2 · resblock_0 · resblock_4
즉 디코더와 후반 ResBlock(5~8)은 tap이 아닙니다. 여기에 넣은 Attention은 PatchNCE가 "직접" 보지 않습니다 — 이게 이 문서의 핵심 주제입니다.

1tap에 Attention을 넣으면 — PatchNCE가 "비교하는 패치 자체"가 바뀐다

PatchNCE는 tap의 특징 → 패치 임베딩 → 유사도 행렬 → 손실 순으로 계산됩니다. Attention이 tap의 특징을 다듬으면 이 행렬이 통째로 바뀝니다. 아래 버튼으로 직접 보세요.

유사도 행렬 (행=출력패치, 열=입력패치)
대각선(흰 테두리)=같은 위치=정답 · 밝을수록 확률↑
이 층의 PatchNCE 손실
·
Attention을 켜면 대각선(정답)이 밝아지고 나머지가 어두워집니다 → 정답 확률↑ → PatchNCE 손실↓. 즉 tap에 넣은 Attention은 "PatchNCE가 정답을 더 쉽게 맞히도록" 패치를 정리해줍니다. = 구조 보존 신호 강화.

2왜 또렷해질까 — 같은 위치 패치를 "더 구별되게" 만든다

Attention 전

잡음(speckle)·불필요한 채널이 섞여 있어, 출력 패치가 입력의 여러 위치와 비슷해 보입니다. → 정답(같은 위치)과 오답이 헷갈림 → 손실 큼.

Attention 후

중요한 구조·채널만 강조되어 각 위치의 특징이 또렷이 구별됩니다. → 정답 위치와만 유사, 다른 위치와는 멀어짐 → 손실 작음.

비유: 흐릿한 졸업사진(Attention 전)에서 친구 찾기는 어렵지만, 또렷한 사진(Attention 후)에선 쉽습니다. Attention은 PatchNCE라는 "짝 찾기 게임"의 사진 해상도를 높여주는 셈입니다.

3반대로, tap이 없는 곳(디코더)에 Attention을 넣으면?

디코더 특징은 PatchNCE가 패치로 읽지 않습니다. 그래서 유사도 행렬을 직접 바꾸지 않습니다. 대신 다른 일을 합니다.

디코더 Attention이 하는 일

  • 출력 이미지의 외형(색·질감·디테일)을 직접 다듬습니다.
  • 주 감독관은 GAN loss(진짜 같은가?)입니다.
  • PatchNCE는 직접 관여하지 않습니다(간접 영향만 — 4장).

그래서 생기는 차이

  • 구조 보존(도로·경계)보다 색 변환·질감 표현에 영향이 큼.
  • 강하게 쓰면 입력에 없던 artifact가 생길 수 있음(통제하는 NCE 신호가 약하므로).
  • 그래서 "encoder+resblocks로 먼저 검증 후 선택적으로" 권장.
정리: tap 위치 Attention = 콘텐츠(구조) 보존을 직접 강화, tap 밖(디코더) Attention = 출력 외형(스타일)을 다듬되 PatchNCE 통제 밖.

4두 갈래 gradient 경로 (애니메이션)

"디코더 Attention은 PatchNCE와 완전히 무관한가?" 아닙니다 — 약하고 간접적인 연결은 있습니다. 생성 이미지(fake_B)가 다시 인코더로 들어가 비교되기 때문입니다.

GAN loss (직접·강함) PatchNCE (간접·약함, fake_B 되먹임)
디코더 Attention → 출력 fake_B에 영향 → ① GAN loss가 굵게 직접 학습(스타일), ② fake_B가 인코더로 되먹여져 PatchNCE에 가늘게 간접 전달. 그래서 "구조 보존" 신호로는 약합니다.

5위치별로 한눈에 — 어떤 Attention이 PatchNCE를 직접 받나 (토글)

체크박스로 Attention을 켜보세요. 분홍 배지=PatchNCE 직접, 회색 배지=tap 밖(GAN 주도·NCE 간접).

6한 장 요약

  • tap에 Attention → PatchNCE가 읽는 패치를 정제 → 유사도 행렬의 대각선↑ → NCE 손실↓ → 구조 보존 직접 강화.
  • tap 밖(디코더·후반 ResBlock)에 Attention → PatchNCE가 직접 안 읽음 → 주로 GAN loss로 출력 외형(색·질감)을 다듬음.
  • 디코더 Attention도 fake_B 되먹임으로 약한 간접 NCE 신호는 받지만, 구조 보존 효과는 작음.
  • 그래서 구조 보존이 목표면 encoder+resblocks(tap), 출력 색/질감 다듬기가 목표면 decoder — 목적에 맞게 위치를 고르세요.
