CUT TRAINING INTERNALS

CUT은 어떻게 학습하는가

Generator(인코더–ResNet–디코더), Discriminator(PatchGAN), 그리고 PatchNCE가 한 스텝 안에서 어떤 순서로 무엇을 업데이트하는지, 각 loss의 의미와 "discriminator가 이미지를 통과시키는 기준"의 진짜 의미를 코드에 근거해 설명합니다.

근거 코드: modules/cut_model.py · modules/losses.py

0. 등장 네트워크 1. 전체 흐름 2. Discriminator 3. GAN loss 4. PatchNCE 4+. NCE 그림 5. 업데이트 순서 6. "통과 기준"의 진실 6+. 학습 곡선 7. 언제 멈추나

0등장하는 4개의 네트워크

CUT은 GAN 한 쌍(G, D)에 더해, PatchNCE를 위한 인코더(E)와 투영 MLP(F)를 함께 학습합니다.

netG · Generator

인코더–ResNet블록–디코더. 입력 SAR real_A → 가짜 optical fake_B 생성. 우리가 최종적으로 원하는 망.

netD · Discriminator

PatchGAN. 입력 이미지가 진짜 optical인지 가짜인지를 패치 단위로 채점. G를 압박하는 "심판".

netE · Encoder

독립 망이 아니라 netG의 앞부분과 가중치를 공유. Generator 중간 특징을 여러 층에서 뽑아내는 도구. (학습 대상 아님 — G와 함께 갱신됨)

netF · PatchSampleMLP

중간 특징의 패치를 작은 2-layer MLP로 임베딩+정규화. PatchNCE 비교를 위한 투영 헤드. NCE loss로 갱신.

핵심: 학습되는 파라미터는 G, D, F 세 개입니다. E는 G의 일부라 따로 갱신되지 않습니다.

1한 스텝의 전체 데이터 흐름

한 학습 스텝마다 SAR 한 장(real_A)과 optical 한 장(real_B)이 들어옵니다. (CUT은 unpaired라 둘은 무관한 장면)

real_A (SAR)   +   real_B (optical) 입력 두 장
netG(real_A) → fake_B (가짜 optical)  |  netG(real_B) → idt_B (항등 출력) cut 모드는 real_A·real_B를 한 번에 넣어 fake_B와 idt_B를 동시에 생성
netD가 fake_B, real_B를 채점 → GAN loss"진짜 같은가?"
netE+netF가 real_A↔fake_B 패치 비교 → PatchNCE loss"내용이 보존됐는가?"
두 loss는 서로 다른 것을 책임집니다. GAN loss는 "결과가 optical 도메인처럼 보이는가(스타일)", PatchNCE는 "입력의 구조·내용이 유지되는가(콘텐츠)". 이 둘이 함께 있어야 'SAR의 구조를 유지한 채 optical로 변환'이 됩니다.

2Discriminator는 어떻게 동작하나 (PatchGAN)

가장 큰 오해 포인트입니다. Discriminator는 "이미지 1장 = 진짜/가짜 한 개의 답"을 내지 않습니다.

스칼라가 아니라 "점수 격자"를 출력

netD는 Conv를 거쳐 N×N 격자를 출력합니다. 각 칸은 입력 이미지의 한 패치(약 70×70 영역, receptive field)가 얼마나 진짜 같은지를 나타내는 값입니다. 즉 "이미지를 여러 패치로 나눠 각각 채점"합니다.

높음 = 진짜(real)에 가깝다고 판단 낮음 = 가짜(fake)로 판단
real_B를 넣었을 때 (이상적 D)
fake_B를 넣었을 때 (이상적 D)

이 코드의 출력 채널은 1이고 sigmoid가 없습니다(LSGAN). 즉 출력은 0~1 확률이 아니라 실수 점수이며, 학습이 "진짜=1, 가짜=0"에 가까워지도록 회귀시킵니다.

# Discriminator: Conv 스택으로 다운샘플 후 1채널 점수맵 출력
x = ConvBlock(64,4,strides=2)(inputs) → 128 → 256 → 512
outputs = ConvBlock(1, 4)(x)          # (N, n, n, 1) 점수 격자, 활성화 없음
왜 패치 단위? 전체 이미지 1개 점수보다, 국소 텍스처의 진짜다움을 패치별로 따지는 편이 고주파 디테일(질감) 품질을 잘 끌어올립니다. 파라미터도 적고 임의 크기 이미지에 적용 가능합니다.

3GAN loss — D와 G의 줄다리기 (LSGAN)

이 구현은 LSGAN(최소제곱)을 씁니다. 즉 점수와 목표값(1 또는 0)의 평균제곱오차(MSE)입니다.

Discriminator 목표 
D_real_loss = MSE(1, D(real_B))   # 진짜는 1로
D_fake_loss = MSE(0, D(fake_B))   # 가짜는 0으로
D_loss = (D_real_loss + D_fake_loss) * 0.5

D는 진짜를 1, 가짜를 0으로 밀어내려 합니다. → 가짜를 잘 골라내도록 학습.

Generator 목표 
G_gan_loss = MSE(1, D(fake_B))    # 가짜를 1로 '속이려'

G는 자기가 만든 가짜에 대해 D가 1(진짜)이라고 답하게 만들려 합니다. → 더 진짜 같은 이미지를 생성.

적대(adversarial) 관계: 같은 D(fake_B)를 두고 D는 0으로, G는 1로 끌어당깁니다. 한쪽이 좋아지면 다른 쪽 loss가 올라가는 제로섬 줄다리기입니다. 그래서 "loss가 0으로 수렴"하지 않습니다(6장에서 상술).

4PatchNCE loss — 내용 보존을 강제하는 대조 학습

GAN만 있으면 "optical처럼 보이지만 입력과 무관한 그림"이 나올 수 있습니다. PatchNCE가 입력의 구조를 출력이 따라가도록 묶어줍니다.

아이디어: "같은 위치 패치는 가깝게, 다른 위치는 멀게"

입력 real_A와 출력 fake_B를 각각 netE로 통과시켜 여러 층의 중간 특징을 얻고, 같은 공간 위치의 패치를 뽑습니다.

  • Query: 출력(fake_B)의 어떤 위치 패치
  • Positive(양성): 입력(real_A)의 같은 위치 패치 — 가까워야 함
  • Negatives(음성): 입력의 다른 위치 패치들 — 멀어야 함

이를 InfoNCE(소프트맥스 교차엔트로피)로 학습합니다. 즉 "query가 자기 짝(positive)을 N개 후보 중에서 골라내는 분류 문제"로 만듭니다.

# losses.py 핵심
logit = (feat_fake · feat_realᵀ) / temperature   # 유사도 행렬
# 대각선 = positive(같은 위치), 나머지 = negative
loss = CrossEntropy(target=대각선=1, logit)        # 대각선이 커지도록
여러 층에서
nce_layers(인코더~ResNet 초반)의 여러 특징에서 동시에 계산해 합산 → 저수준~고수준 내용 모두 보존
temperature
0.07. 작을수록 positive/negative 구분을 더 날카롭게(어렵게) 만듦
netF의 역할
패치를 MLP로 임베딩+L2정규화해 코사인 유사도 비교가 의미 있게 만듦

Identity NCE (cut 모드)

추가로 real_B → idt_B 에 대해서도 NCE를 겁니다. optical을 넣으면 거의 그대로 나와야 하므로, 이는 색·내용을 함부로 바꾸지 않게 하는 정규화입니다. 최종 NCE_loss = (NCE(real_A,fake_B) + NCE(real_B,idt_B)) / 2.

PatchNCE는 "통과/탈락" 판정이 아닙니다. 매 스텝 조금씩 낮아지도록 minimize되는 연속 값이며, 낮을수록 입력↔출력 위치 대응이 또렷하다는 뜻입니다. 특정 임계값을 넘기면 끝나는 게 아닙니다.

4+PatchNCE 한눈에 보기 — query · positive · negative

아래는 특징맵을 6×6 패치 격자로 단순화한 그림입니다. 출력의 한 패치(query)가 입력에서 자기 짝을 찾아내는 분류 문제로 학습됩니다.

입력 real_A (SAR) 특징
positive (같은 위치) negative (다른 위치)
⟷
출력 fake_B (생성) 특징
query (이 패치 기준)
파란 query 패치는 입력의 같은 위치(초록 positive) 와는 가까워지고, 나머지 위치(빨강 negative) 와는 멀어지도록 학습됩니다. 이게 모든 위치에 대해 일어나면, 출력의 각 부분이 입력의 대응 부분을 닮게 되어 구조가 보존됩니다.

InfoNCE를 "객관식 문제"로 이해하기

query 1개에 대해 후보 = {정답 1개(positive) + 오답 여러 개(negatives)}. 모델은 유사도 점수에 softmax를 씌워 정답을 고르도록 학습합니다(교차엔트로피).

유사도 = query · 후보 / temperature(0.07)
정답(positive) 점수는 ↑, 오답(negative) 점수는 ↓ 가 되도록 cross-entropy 최소화
→ 같은 위치 패치쌍의 상호정보(mutual information)를 최대화

temperature가 작을수록(0.07) 점수 차이를 더 크게 벌려 "헷갈리는 오답"도 확실히 밀어냅니다.

5한 스텝 안의 업데이트 순서 (코드 그대로)

하나의 tf.GradientTape로 모든 loss를 계산한 뒤, D → G → F 순서로 각자의 파라미터만 갱신합니다.

1
순전파 — fake = netG(real) 로 fake_B, idt_B 생성
2
D loss 계산 — D(fake_B)는 0으로, D(real_B)는 1로 (MSE)
3
G loss 계산 — GAN(가짜를 1로) + PatchNCE(+ identity NCE)(+옵션 grad/color)
4
D 갱신 D_grads = tape.gradient(D_loss, netD.vars) → D optimizer
5
G 갱신 G_grads = tape.gradient(G_loss, netG.vars) → G optimizer
6
F 갱신 F_grads = tape.gradient(NCE_loss, netF.vars) → F optimizer
업데이트 대상사용하는 loss역할
netDD_loss (GAN, real+fake)진짜/가짜 더 잘 구분
netGG_loss = GAN + NCE (+grad+color)더 진짜같고 내용 보존된 이미지 생성
netFNCE_loss대조 학습에 좋은 패치 임베딩 학습
중요: 각 loss는 자기 네트워크만 갱신합니다. 예컨대 NCE_loss의 그래디언트는 G와 F로 흐르지만, D_loss는 D만 바꿉니다. 그래서 "G가 D를 직접 학습시키는" 일은 없습니다 — 서로 독립적으로 자기 목표만 최적화합니다.

6"loss가 어느 값 이하면 D가 통과시킨다"? — 그런 임계값은 없습니다

질문 주신 핵심 오해를 바로잡는 부분입니다.

GAN에는 "이 값 이하면 완성" 같은 고정 임계값이나 합격선이 없습니다. Discriminator는 고정된 시험관이 아니라, G와 동시에 계속 강해지는 상대입니다. G가 좋아지면 D도 더 깐깐해지므로, "특정 loss에 도달하면 D가 pass" 하는 종료 조건이 성립하지 않습니다.

그럼 무엇을 향해 가나? — 내쉬 균형(Nash equilibrium)

이론적 이상점은 "G가 만든 가짜가 진짜와 구분 불가"가 되어, D가 어느 패치든 확신하지 못하는 상태입니다.

  • LSGAN에서 이상적 균형의 D 출력은 진짜·가짜 모두 0.5 부근 (1도 0도 아님).
  • 이때 D_loss는 0이 아니라 중간값에서 정체하고, G_gan_loss도 0으로 안 갑니다.
  • 즉 "loss가 0에 수렴 = 좋은 것"이 아닙니다. 한쪽 loss가 0으로 가면 보통 학습이 무너진 신호(mode collapse 등)입니다.
올바른 그림: D_loss와 G_loss가 어느 중간 범위에서 서로 밀고 당기며 안정적으로 진동하고, 그 사이 PatchNCE loss가 꾸준히 낮아지며, 생성 이미지 품질이 점점 좋아지는 것 — 이게 건강한 학습입니다.

loss 값을 읽는 법 (직관)

관찰해석
D_loss가 0에 가깝게 떨어짐D가 G를 압도(가짜 다 간파). G가 못 배움 → 불균형
G_gan_loss가 0에 가깝게 떨어짐G가 D를 너무 쉽게 속임. D가 약함 → 품질 신뢰 어려움
둘이 중간값에서 진동 + NCE 하락✅ 건강한 적대적 균형, 품질 향상 중
NCE가 안 떨어짐입력 구조가 출력에 반영 안 됨 → 내용 붕괴 위험

6+학습 곡선 예시 — 건강한 학습 vs 붕괴

아래는 전형적인 패턴을 그린 예시(개념도)입니다. 실제 값 범위는 데이터/설정마다 다르지만 모양의 의미는 동일합니다.

✅ 건강한 학습

  • D_loss / G_loss: 0으로 안 가고 중간 범위에서 진동·공존
  • NCE_loss: 꾸준히 우하향 (내용 보존 개선)
  • → 이미지 품질이 점점 향상

⚠️ 붕괴 (D가 압도)

  • D_loss → 0 으로 급락 (가짜를 다 간파)
  • G_loss 가 치솟음 (D를 못 속임 → 그래디언트 약화)
  • NCE 정체 → 학습 정지/모드붕괴 신호
D_loss G_loss NCE_loss
대처: 붕괴가 보이면 — D가 너무 강하면 G 학습률↑/D 학습률↓, 또는 lambda_NCE·구조손실 조정. 한쪽 loss가 0으로 수렴하는 건 "잘됨"이 아니라 경고 신호로 읽으세요.

7그러면 학습은 언제 끝내나

loss 임계값이 아니라 아래 기준으로 멈춥니다.

  • 정해진 epoch 수까지 학습 (이 코드는 기본 400 epoch, GUI에서 조정).
  • 생성 이미지의 정성 평가: epoch별 저장 이미지(Source/Translated/Target/Identity)를 눈으로 비교.
  • 정량 지표: paired GT가 없으면 FID/KID(분포 거리), downstream task 성능 등으로 측정.
  • 안정성 모니터링: D_loss/G_loss가 한쪽으로 붕괴하지 않는지, NCE가 꾸준히 낮아지는지.
정리하면 — Discriminator는 "합격 도장"을 찍는 장치가 아니라 G를 계속 끌어올리는 적대적 코치입니다. 완료 판단은 사람/지표(FID 등)가 하지, D의 점수 한 컷으로 정해지지 않습니다.

★한 장 요약

  • G가 SAR→optical 변환, D(PatchGAN)가 패치별 진짜/가짜 채점, F가 PatchNCE용 임베딩.
  • 한 스텝: 순전파 → D_loss·G_loss 계산 → D, G, F 순서로 각자 갱신.
  • GAN loss = 스타일(optical다움), PatchNCE = 콘텐츠(입력 구조 보존). 둘 다 필요.
  • PatchNCE는 임계값 통과가 아니라 계속 minimize되는 연속 값.
  • "loss가 X 이하면 D가 통과"는 없음. D와 G는 균형점에서 진동하며, 완료는 epoch/FID/육안으로 판단.
