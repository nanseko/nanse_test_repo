PatchNCE Loss, 그림으로 이해하기

CUT의 핵심 손실인 PatchNCE가 정확히 어떻게 계산되고, 무엇을 의미하는지를 애니메이션과 인터랙티브 요소로 한 단계씩 설명합니다. 슬라이더를 직접 움직여 보세요.

0PatchNCE는 무슨 문제를 푸는가

CUT은 짝이 없는(unpaired) 변환입니다. SAR 입력에 대응하는 "정답 optical"이 없습니다. 그래서 "출력을 정답과 픽셀 비교"가 불가능합니다.

GAN loss만 쓰면 "optical처럼 보이긴 하는데 입력과 전혀 다른 그림"이 나올 수 있습니다(내용 붕괴). 이를 막으려면 "출력이 입력의 구조·내용을 유지"하도록 강제하는 장치가 필요합니다.

PatchNCE = 입력과 출력의 "같은 위치 조각(patch)"이 서로 닮도록 묶어주는 끈. 정답 이미지 없이도, 입력 자신을 기준으로 내용 보존을 학습합니다.

1한 줄 비유 — "짝 찾기 게임"

출력 이미지의 한 조각(query)을 들고, 입력 이미지의 여러 조각들 사이에서 "내 원래 자리(같은 위치)"를 찾아내는 객관식 문제를 푸는 것이 PatchNCE입니다.
정답(같은 위치 = positive)은 가깝게, 오답(다른 위치 = negative)은 멀게 만들면, 출력의 각 부분이 입력의 대응 부분을 닮게 되어 구조가 보존됩니다.

2전체 계산 흐름 (6단계)

아래 "▶ 단계별 재생"을 누르면 한 단계씩 켜집니다.

① 특징 추출 →② 같은 위치 패치 샘플 →③ MLP 투영 + 정규화 →④ 유사도 행렬 →⑤ InfoNCE(softmax) →⑥ 여러 층 평균
재생 버튼을 눌러보세요.

3Step ②: "같은 위치" 패치를 뽑는다

입력(real_A)과 출력(fake_B)의 특징맵에서 완전히 동일한 위치의 패치들을 무작위로 N개 뽑습니다. (코드의 patch_ids가 두 쪽에 동일하게 쓰여 "같은 자리"가 보장됩니다.)

입력 real_A 특징맵
같은 위치 →
출력 fake_B 특징맵
입력에서 뽑힌 위치 출력에서 같은 위치
한 번에 모든 픽셀을 비교하면 너무 무겁기 때문에, 매 스텝 무작위로 N개 위치만 골라 비교합니다(코드 기본 256개). 그래서 이름이 "Patch"NCE입니다.

4Step ③: 패치를 "임베딩(벡터)"으로 — 그리고 학습의 의미

각 패치를 작은 2-layer MLP(netF)에 통과시키고 L2 정규화합니다. 그러면 모든 패치가 "단위 구(반지름 1) 위의 점"이 됩니다. 점들 사이 각도(코사인 유사도)가 곧 닮은 정도입니다.

입력 패치 임베딩 (고정) 출력 패치 임베딩 (학습으로 이동) positive 쌍 연결선
애니메이션: 학습이 진행될수록 출력 패치(파랑)가 자신의 짝(positive, 분홍)에게 끌려가 가까워집니다. = "같은 위치끼리 닮아진다". 이게 PatchNCE가 실제로 하는 일입니다.

5Step ④: 유사도 행렬 만들기 (인터랙티브)

출력 패치 N개 × 입력 패치 N개의 모든 쌍의 유사도를 한 표에 담습니다. 행 = 출력 패치(query), 열 = 입력 패치. 대각선(같은 위치)이 positive, 나머지는 negative입니다.

6Step ⑤: InfoNCE = "객관식 정답 맞히기"

유사도 행렬의 각 행에 softmax를 씌워 확률로 바꾸고, "정답(대각선)을 골랐는가"를 교차엔트로피로 채점합니다.

수식 (한 query 기준)

logit_j = (query · 입력패치_j) / τ
p_j = softmax(logit)_j           # 후보들에 대한 확률
loss = −log( p_정답 )            # 정답=같은 위치(대각선)

정답 확률 p_정답이 1에 가까우면 loss→0, 0에 가까우면 loss→∞. 즉 "정답을 확신할수록" 손실이 작아집니다.

왜 "정규화 + temperature"인가

  • L2 정규화: 벡터 크기를 1로 맞춰 방향(각도)만 비교 → 코사인 유사도가 됨.
  • τ(temperature): 유사도 차이를 확대/축소. 작을수록 정답·오답을 더 극단적으로 구분(어려운 오답도 확실히 밀어냄).
  • negative: 같은 이미지 내 다른 위치 패치들. "내 자리"를 다른 자리와 구별하게 함.
객관식 비유: query는 학생, 후보 패치들은 보기. 정답은 "같은 위치" 한 개. softmax는 학생이 매긴 보기별 확신도, cross-entropy는 "정답에 얼마나 확신했나"로 매기는 점수입니다.

7temperature가 loss에 주는 영향 (직접 실험)

위 5절의 슬라이더와 연동됩니다. 같은 유사도라도 τ에 따라 loss가 크게 달라집니다.

τ 값분포 모양정답 확률loss 경향
작음 (0.02~0.07)매우 날카로움 (정답에 집중)높음 → 1에 근접작아짐(정답이 최고 유사도일 때)
기본 (0.07)적당히 날카로움—CUT 기본값
큼 (0.5~1.0)평평함 (다 비슷하게)낮음 → 1/N 근처커짐(ln N 근처)
주의: τ를 줄인다고 "실제로 더 잘 배운" 게 아닙니다. τ는 난이도 조절 다이얼입니다. 정답이 진짜로 가장 닮았을 때만 작은 τ가 loss를 낮춰줍니다. 정답이 안 닮았는데 τ만 줄이면 loss가 오히려 폭증합니다.

8실제 코드와 1:1 매핑 (losses.py)

def __call__(self, source, target, netE, netF):
    feat_source = netE(source)              # ① 입력(real_A) 다층 특징
    feat_target = netE(target)              # ① 출력(fake_B) 다층 특징

    # ② 같은 위치 패치 샘플 + ③ MLP 투영 + L2 정규화
    feat_source_pool, sample_ids = netF(feat_source, patch_ids=None)
    feat_target_pool, _          = netF(feat_target, patch_ids=sample_ids)  # 같은 위치!

    total = 0.0
    for feat_s, feat_t in zip(feat_source_pool, feat_target_pool):   # ⑥ 층마다
        n, dim = feat_s.shape
        logit = (feat_t @ feat_sᵀ) / temp        # ④ 유사도 행렬 (n×n)
        target = eye(n)                          #   대각선=정답(positive)
        loss = cross_entropy(target, logit)      # ⑤ InfoNCE
        total += mean(loss)
    return total / num_layers                    # ⑥ 층 평균
코드의미
netE(source/target)Generator와 가중치 공유하는 인코더로 다층 특징 추출
netF(..., patch_ids)같은 위치 패치 N개를 뽑아 MLP 임베딩 + L2 정규화
feat_t @ feat_sᵀ / temp모든 출력·입력 패치쌍 코사인 유사도를 τ로 나눔
eye(n) (대각선)"같은 위치"가 정답이라는 라벨
cross_entropy각 행이 자기 대각선을 고르도록 학습
/ num_layers여러 층(저수준~고수준) 손실의 평균
"여러 층"의 의미: 인코더 앞쪽 층은 에지·질감 같은 저수준, 뒤쪽 층은 더 의미적인 구조를 담습니다. 여러 층에서 동시에 PatchNCE를 걸어 세밀한 디테일부터 큰 구조까지 모두 보존합니다.

9PatchNCE 값을 읽는 법 & 자주 묻는 것

상황의미
NCE_loss가 꾸준히 ↓✅ 출력이 입력 구조를 점점 잘 따라감 (정상)
NCE_loss가 안 떨어짐/정체내용 보존 실패 위험 — 구조가 무너질 수 있음
NCE가 0에 가깝게 급락대개 출력이 입력을 거의 복사(변환이 약함)하는 신호일 수 있음

자주 묻는 것

Q. positive/negative에 정답 optical이 쓰이나요?
아니요. 입력(real_A)과 그 출력(fake_B)만 씁니다. 정답 optical은 등장하지 않습니다 — 그래서 unpaired에서도 동작합니다.

Q. "PatchLoss"가 따로 있나요?
이 구현에서 패치 기반 손실은 PatchNCELoss 하나입니다(아이덴티티 경로용으로 한 번 더 계산될 뿐). "PatchNCE = PatchNCE Loss"로 보면 됩니다.

Q. GAN loss와 무슨 차이?
GAN = "optical처럼 보이는가(스타일)", PatchNCE = "입력 내용을 유지하는가(콘텐츠)". 둘은 역할이 다르고 함께 작동합니다.

한 줄 요약: PatchNCE = 입력·출력의 같은 위치 패치를 임베딩해, "정답(같은 위치)을 골라내는 객관식"을 풀게 만드는 대조 손실. 정답이 가까워질수록 loss가 작아지고, 결과적으로 구조가 보존됩니다.
